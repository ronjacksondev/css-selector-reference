CSS Selector Reference

This is a reference list for some of the selectors used in HTML styling.

Disclaimer: This document was made while learning and may contain mistakes.

Simple Selectors

Simple selectors are the building blocks of css selector syntax.

Name Example Description
Universal Selector * Selects all elements on the page
Type Selector div Selects all elements of the defined type.
Attribute Selector [href] Selects all elements with the defined attribute.
Class Selector .example Selects all elements with the defined class name.
ID Selector #example Selects the element with the defined ID.

Selector list

Any selectors can be grouped together in a list to share style among several selectors.

Name Example Description
Basic Selector List h1, h2, h3 Selects all h1 or h2 or h3 elements.

Compound Selectors

Compound selectors are simple selectors that are concatenated directly together.

Below are a few examples of possible combinations.

Name Example Description
Type and Attribute input[type="text"] Selects all input elements with the type attribute and with a value set to "text."
Type and Class div.special Selects all div elements with the .special class
Attribute and Class [src].special Selects all elements with an src attribute and the .special class.
Attribute and Attribute [width][height] Selects all elements with a width and height attribute.
Class and Class .special.attention Selects all elements with a .special class and a .attention class

Complex Selectors

Complex selectors are a cobmination of 2 or more selectors joined by a combinator (like an operator).

This is essentially a list of combinators.

Name Example Description
Descendant Combinator p span Selects all span elements that are descendants of p elements.
Child Combinator p > span Selects all span elements that are a direct child of a p element.
Next-sibling Combinator p + span Selects all span elements that immediately follow a p element sibling.
Subsequent-sibling Combinator p ~ span Selects all span elements that have a preceding p element sibling.

Pseudo Selectors

Pseudo selectors select elements by either their state or by other information that is hard to express with simple selectors.

Name Example Description
User Action
:hover a:hover Selects all anchor elements when the pointer hovers over them.
:focus input:focus Selects the input that has screen focus.
Input
:checked input:checked Selects all input elements that are checked = true
:disabled input:disabled Selects all input elements that are disabled = true
:enabled input:enabled Selects all input elements that are disabled = false
:required input:required Selects all input elements that are required = true
:optional input:optional Selects all input elements that are required = false
Tree Structure
:root :root Selects the root element, usually <html>.
Often used in ruleset to store css variable declarations.
:nth-child(n) tr:nth-child(even) Selects every other table row.
:first-child tr:first-child Selects the first table row.
:last-child tr:last-child Selects teh last table row
Location
:visited a:visited Selects all anchor links that the browser has visited.
:any-link a:any-link Selects all anchor tags with an href.
Equivalent to :is(:link, :visited).
Functional
:is() :is(ul, ol) Selects all elements that are an ul or ol.
Allegedly can be faster than a selector list (ul, ol {...} in complex selectors.
:where() :where(ul, ol) Selects all elements that are an ul or ol.
Same as :is() but :where() has zero specificity.
:not() :not(p) Selects any element that is not a p element.
:has() p:has(span.special) Selects any p element that contains a span with .special

Pseudo Elements

Pseudo elements select virtual components of an element.

Pseudo elements allow adding content and styling.

Name Example Description
::before p::before Selects a virtual element before any p element.
::after p::after Selects a virtual element after any p element.
::first-letter p::first-letter Selects the first letter of any p element.
